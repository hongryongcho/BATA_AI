## 1. 개요

LOC(Limit-On-Close) 기반 계단식 분할 매수/매도 알고리즘.  
Fear & Greed 지수와 52주 전고점 대비 낙폭을 조건으로 매수·매도 배수를 동적으로 조정한다.  
백테스트 파라미터는 Google Sheets에서 설정하며, 결과 데이터도 Google Sheets로 출력한다.

| 항목 | 기본값 | 설명 |
|------|--------|------|

| 레버리지 여부 | 자동 감지 | 3배수 ETF는 임계값 ×3 적용 |

## 3. 단위 수량 (Unit Size) 계산

### 3.1 최초 사이클 (Cycle 1)
```
unit_qty = floor(total_capital / N / 최초_진입_가격)
```
- `N` : 등분 수 (기본값: 40, Google Sheets에서 변경 가능)
- `최초_진입_가격` : 첫 매수 실행 당일 종가
- **unit_qty 는 사이클 내내 고정** (주가 변동 무관)

### 3.2 사이클 리셋
```
새_total_capital = 현금 + (보유주식 × 당시종가) + 실현손익 누적
unit_qty = floor(새_total_capital / N / 신규_진입_가격)
```
- 보유 포지션이 0으로 청산될 때 사이클 종료
- 다음 매수 시점에 새로운 unit_qty 를 재계산

---

## 4. 매매 실행 규칙

### 4.1 실행 시점
- **하루 1회**, 당일 종가 기준 LOC 주문
- 조건 판단은 당일 종가 확정 후 수행

### 4.2 기본 흐름 (Basic Cycle)

```
if 보유주식 == 0:
    → 1회 매수 (unit_qty 만큼)
elif 현재가 >= 평단 × (1 + 기본수익기준%):
    → 1회 매도 (unit_qty 만큼)
else:
    → 1회 매수 (unit_qty 만큼)
```

**기본 수익 기준:**
- 1배수 주식: **+3%** (평단 대비)
- 3배수 ETF: **+9%** (평단 대비)

---

## 5. 매수 배수 (Buy Multiplier)

52주 전고점 대비 현재가 낙폭에 따라 매수 수량을 증가.

### 5.1 1배수 주식 (SPY, QQQ, SOXX 등)

| 52주 전고점 대비 낙폭 | 매수 수량 |
|----------------------|----------|
| 0% ~ -9.99% | 1× unit_qty |
| -10% ~ -14.99% | 2× unit_qty |
| -15% ~ -19.99% | 3× unit_qty |
| -20% 이상 | 4× unit_qty |

### 5.2 3배수 레버리지 ETF (SPXL, TQQQ, SOXL 등)

| 52주 전고점 대비 낙폭 | 매수 수량 |
|----------------------|----------|
| 0% ~ -29.99% | 1× unit_qty |
| -30% ~ -44.99% | 2× unit_qty |
| -45% ~ -59.99% | 3× unit_qty |
| -60% 이상 | 4× unit_qty |

> **임계값 공식:** `leverage_multiplier × 일반_임계값`  
> 현재 설정: 3× ETF → 10%×3=30%, 15%×3=45%, 20%×3=60%

---

## 6. 매도 배수 (Sell Multiplier)

평단 대비 수익률에 따라 매도 수량을 증가.

### 6.1 1배수 주식

| 평단 대비 수익률 | 매도 수량 |
|----------------|----------|
| 손실 ~ +9.99% | 1× unit_qty |
| +10% ~ +19.99% | 2× unit_qty |
| +20% ~ +29.99% | 3× unit_qty |
| +30% 이상 | 4× unit_qty |

### 6.2 3배수 레버리지 ETF

| 평단 대비 수익률 | 매도 수량 |
|----------------|----------|
| 손실 ~ +29.99% | 1× unit_qty |
| +30% ~ +59.99% | 2× unit_qty |
| +60% ~ +89.99% | 3× unit_qty |
| +90% 이상 | 4× unit_qty |

> 수량 제한: `min(매도배수 × unit_qty, 보유수량)`

## 7. 특수 조건

### 7.1 갭업 강제 매도
```
if (당일종가 / 전일종가 - 1) >= gap_up_pct:
    → 현재 수익 무관, 손실 구간이라도 1회 매도 (unit_qty)
```
- 1배수: `gap_up_pct = 2%`
- 3배수: `gap_up_pct = 6%`

### 7.2 Fear & Greed 필터

| 구간 | 허용 동작 |
|------|-----------|
| Extreme Fear (0~24) | **매수만** 허용 (매도 금지) |
| Fear ~ Greed (25~74) | 제한 없음 |
| Extreme Greed (75~100) | **매도만** 허용 (매수 금지) |

- Fear & Greed 데이터 소스: CNN Fear & Greed Index API
- 갱신: 매일 1회

### 7.3 현금 소진
```
if 잔여현금 < 당일종가 × 매수수량:
    → 매수 건너뜀, 다음 거래일로 이월
```

---

## 8. 사이클 종료 및 리셋

```
if 보유주식 == 0 AND 매도 완료:
    사이클 종료
    new_capital = 현금잔고 + 실현손익_누적
    unit_qty = floor(new_capital / N / 다음매수시_종가)
    사이클 카운터 += 1
```

```bash
cd 03_BATA_ALGO
python3 run_backtest.py
```

1. Google Sheets [Summary] 시트에서 파라미터 읽기
2. Yahoo Finance에서 주가 데이터 다운로드
